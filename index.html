Cyclistic Bike Share Analysis

Case Study: Converting Casual Riders to Annual Members

Tools used: Excel, BigQuery SQL, and Tableau

Project Overview

This project analyzes Cyclistic bike-share data to understand how casual riders and annual members use the service differently. The objective is to uncover behavioral patterns and generate data-driven insights that help convert casual riders into profitable annual memberships.

  • Riding patterns between casual riders and members
  • Trip duration trends across different user types
  • Weekday vs weekend usage behavior
  • Marketing opportunities to convert casual riders

Business Problem

Cyclistic wants to increase the number of annual memberships.

The marketing team wants to understand:

  • How casual riders and members behave differently
  • When casual riders ride the most
  • How to convert casual riders into members

Dataset

The dataset contains bike-share trip records including:

  • ride_id
  • rideable_type
  • start_time
  • end_time
  • station names
  • usertype (Customer / Subscriber)

Data Pipeline

Data Cleaning in BigQuery

A cleaned view was created to prepare the dataset for analysis.


CREATE OR REPLACE VIEW `primal-chariot-479401-i4.divvy_trip_2019_q4.v_trips_2019_clean` AS

SELECT
*,

TIMESTAMP_DIFF(end_time,start_time,MINUTE) AS ride_length,

FORMAT_DATE('%A',DATE(start_time)) AS day_of_week

FROM `primal-chariot-479401-i4.divvy_trip_2019_q4.trips_2019_all`
WHERE end_time > start_time;

SQL Analysis

Average Ride Duration by User Type


CREATE OR REPLACE TABLE trend_avg_ride_by_usertype_day AS

SELECT
day_of_week,

CASE
WHEN usertype='Subscriber' THEN 'member'
WHEN usertype='Customer' THEN 'casual'
END AS member_type,

AVG(duration_sec) AS avg_duration_sec

FROM trips_2019_clean
GROUP BY day_of_week,member_type;

Total Rides by Day


CREATE OR REPLACE TABLE summary_day AS

SELECT
day_of_week,
COUNT(*) AS total_rides,
AVG(duration_sec) AS avg_ride_sec

FROM trips_2019_clean
GROUP BY day_of_week;

Ride Count by Member Type


CREATE OR REPLACE TABLE summary_member_day AS

SELECT
usertype AS member_type,
day_of_week,
COUNT(*) AS total_rides,
AVG(duration_sec) AS avg_ride_sec

FROM trips_2019_clean
GROUP BY member_type,day_of_week;

Member vs Casual Comparison


CREATE OR REPLACE TABLE trend_member_vs_casual_users AS

SELECT

CASE
WHEN usertype='Subscriber' THEN 'member'
WHEN usertype='Customer' THEN 'casual'
END AS member_type,

COUNT(*) AS total_rides

FROM trips_2019_clean
GROUP BY member_type;

Weekday Ride Trends


CREATE OR REPLACE TABLE trend_weekday_ride_by_member_casual AS

SELECT
day_of_week,
usertype,
COUNT(*) AS total_rides

FROM trips_2019_clean
GROUP BY day_of_week,usertype;

Most Common Ride Day


CREATE OR REPLACE TABLE trips_2019_summary AS

SELECT
AVG(duration_sec) AS avg_duration_sec,
MAX(duration_sec) AS max_duration_sec,

(
SELECT day_of_week
FROM trips_2019_clean
GROUP BY day_of_week
ORDER BY COUNT(*) DESC
LIMIT 1
) AS mode_day

FROM trips_2019_clean;

Overall Ride Summary


CREATE OR REPLACE TABLE summary_overall AS

SELECT
COUNT(*) AS total_rides,
AVG(duration_sec) AS avg_ride_sec,
MIN(duration_sec) AS min_ride_sec,
MAX(duration_sec) AS max_ride_sec

FROM trips_2019_clean;

Key Insights

  • Casual riders peak on weekends.
  • Members ride more during weekdays.
  • Casual riders have longer ride durations.
